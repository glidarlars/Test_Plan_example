# 4.9 Test Deliverables

The test project delivers the following artifacts to the main Project **A1**:

| Document / Deliverable | When / Where to be Found & Delivered |
| :--- | :--- |
| **Test Plan** *(this document & references)* | Reviewed and updated prior to each project checkpoint; stored in **Pythia**. |
| **Incident Reports** *(defects/bugs)* | Reported into project-specific tools (Jira) as soon as an issue is identified. |
| **Sprint Test Reports** | For software sprint development, a concise summary report is generated after each sprint and saved in **Confluence**. |
| **Milestone Test Reports** | Compiled and reviewed prior to each technical review of the project; saved in **Pythia**. |
| **Test Report** *(execution logs)* | Logs for each executed test case are uploaded into the test report database: **Pythia**. |
| **Test Specifications** | Written and maintained in the test specification database: **Pythia**. |
| **Statement of Compliance** | Embedded in each milestone summary test report, establishing compliance with the product requirements. |
